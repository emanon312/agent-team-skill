# 单一代码审查模板

> 适用场景：代码审查/Bug 排查/方案分析/文档检查 | 模式：Single Agent

## 角色速查

| 任务类型 | 推荐 agent | 审查重点 |
|----------|-----------|----------|
| 代码审查 | 红小审 | 安全性、可读性、性能、最佳实践、潜在 bug |
| Bug 排查 | 绿小验 | 根因分析、复现步骤、修复建议、影响范围 |
| 方案分析 | 红小谋 | 优缺点、风险、替代方案、实施成本 |
| 文档检查 | 红小文 | 准确性、可读性、完整性、格式一致性 |

## 依赖关系

无（单 agent）。

## 典型验收标准

- 代码审查：列出至少 3 个具体问题（含行号/位置）+ 改进建议
- Bug 排查：给出根因 + 复现步骤 + 修复方案（如能确定）
- 方案分析：优劣势对比 + 风险评估 + 推荐结论
- 文档检查：列出错误/遗漏/不清晰处，逐条标注

## 预估耗时

| 角色 | 软超时 | 硬超时 |
|------|--------|--------|
| 全部 | 5 min | 10 min |

## 派出 prompt 模板

```
你是<角色>。请<任务>：
- 审查范围：<文件/代码段>
- 重点：<审查维度>
- 输出格式：逐条列出发现的问题 + 改进建议

完成后用消息（SendMessage）把结果发回 main。
```
