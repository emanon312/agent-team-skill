# 模板格式规范

## 模板是什么

模板是**一份可复制的分工参考**——当用户的任务与模板匹配时，指挥官可以直接参考模板的花名册和角色配置，省去调研时间。

## 模板不是死规矩

- 模板是**起点**，不是终点。领域相同的任务，技术栈、团队规模不同，角色可能需要增删。
- 如果模板和实际需求有出入，**以调研结果为准**（步骤 0 的 RESEARCH.md 优先级高于模板）。
- 模板主要解决"这个领域我不知道该怎么拆"的冷启动问题。

## 模板文件结构

```markdown
# <模板名称>

> 适用场景：<一句话描述> | 模式：Full Team / Quick Research / Single Agent

## 花名册（或角色速查）

> Full Team / Quick Research 模式用「花名冊」表列出所有成员；Single Agent 模式用「角色速查」表列出不同任务对应的推荐 agent。

| 中文名 | 角色 | 职责 | 建议 skills |
|--------|------|------|------------|
| 红小X  | ...  | ...  | ...        |

## 依赖关系

```
角色A → 角色B → 角色C
```
（无依赖标注"全并行"）

## 典型验收标准

- 角色A：<可验证条件>
- 角色B：<可验证条件>

## 预估耗时

| 角色 | 软超时 | 硬超时 |
|------|--------|--------|
| ...  | ...    | ...    |

## 来源

> 此模板基于以下实战/调研沉淀：
> - <日期>：<来源任务描述>
```

## 模板存放位置

```
.agent-team/templates/
├── TEMPLATE-FORMAT.md    ← 本文件
├── <模板1>.md
├── <模板2>.md
└── ...
```

## 模板索引

模板索引在 `SKILL.md` 末尾的 `## 模板索引` 节中维护，格式：

```markdown
| 模板 | 领域 | 模式 | 角色数 | 来源 |
|------|------|------|--------|------|
| `game-2d-godot.md` | Godot 2D 游戏 | Full Team | 4 | 2026-06-21 Godot 2D 平台游戏调研 |
```

每新增一个模板，同步更新索引表。
